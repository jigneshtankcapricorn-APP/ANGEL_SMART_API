  WHAT IS THIS PROJECT?
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

This is a stock trading dashboard built with:
  - Python
  - Streamlit (web app framework)
  - Angel One SmartAPI (live stock data from Angel One broker)
  - Hosted on Streamlit Cloud (free hosting connected to GitHub)

The app has 5 pages:
  1. Login       — Custom username/password login
  2. Dashboard   — Nifty 50, Bank Nifty, Nifty 500 live index data
  3. Watchlist   — Scan 2,242 NSE stocks for stocks near 52W Low
  4. Signals     — SuperTrend(10,3) buy/watch signals
  5. Sector Trends — Live NSE sector index performance

Credentials are NEVER stored in GitHub code.
They are stored in Streamlit Cloud Secrets only.

To set them:
  Streamlit Cloud → Your App → Settings → Secrets → Paste this:

  APP_USERNAME = "your_username"
  APP_PASSWORD = "your_password"
  API_KEY      = "your_angel_one_api_key"
  CLIENT_ID    = "your_angel_one_client_id"
  PASSWORD     = "your_angel_one_trading_password"
  TOTP_KEY     = "your_totp_base32_secret"

  APP_USERNAME / APP_PASSWORD = the login you created yourself for the app
  API_KEY / CLIENT_ID / PASSWORD / TOTP_KEY = from Angel One SmartAPI account

1. LOGIN PAGE ADDED
   - New login page with username + password
   - Credentials stored in Streamlit Secrets (NOT in GitHub)
   - All pages redirect to login if not authenticated
   - Logout button in sidebar

2. FULL STOCK LIST — 2,242 STOCKS
   - Old app had only 500 stocks
   - Excel file (Untitled_spreadsheet.xlsx) had 2,242 NSE stocks
   - All 2,242 stocks now loaded from Excel into data/stocks_full.csv
   - CSV only stores: name, symbol, market_cap_cr (nothing else)

3. ALL PRICES 100% LIVE FROM ANGEL ONE
   - Old app was reading 52W Low, 52W High, LTP from CSV (stale data)
   - Now everything is fetched live from Angel One API:
       * LTP (current price)
       * 52W High (from 365-day history)
       * 52W Low (from 365-day history)
       * Volume (today)
       * 20-day average volume
   - CSV is used ONLY for stock name and symbol

4. DATA PERSISTENCE — NO MORE DATA LOSS
   - Old app wiped watchlist/signals data every time you changed page
   - New app uses st.cache_resource (server memory)
   - Data survives: page navigation, browser refresh
   - Data clears only when: you click Clear Data, or Streamlit server restarts
   - Both Watchlist and Signals show timestamp of last fetch
   - Manual refresh only — no auto-refresh

5. SECTOR TRENDS — FIXED WITH REAL DATA
   - Old app showed wrong/fake sector data (was estimating from 52W Low %)
   - New app fetches REAL NSE Sector Index OHLC from Angel One
   - 20 sector indices: Nifty IT, Nifty Bank, Nifty Pharma, Nifty Auto, etc.
   - Shows real Daily %, Weekly %, Monthly % from actual index prices
   - Falls back to stock-based estimate if index fetch fails

6. BATCH SCANNING ADDED (Watchlist + Signals)
   - Scanning all 2,242 stocks takes 4-5 hours (one by one API calls)
   - Batch mode lets you scan 200 at a time (~25 min per batch)
   - Results from each batch merge automatically
   - If timeout occurs, set "Start from stock #" to resume
   - After each batch, start number auto-advances to next batch

7. MARKET CAP FILTER ADDED (Watchlist)
   - Enable/disable checkbox
   - Manual input in Crores
   - Quick select: Small Cap / Mid Cap / Large Cap / All
   - Note: market cap known for ~499 stocks. Other 1,743 pass through.

8. FILTER SETTINGS PERSIST ACROSS PAGES
   - Old app reset all filters when you navigated away
   - Now all settings remembered in session_state:
       * Min % / Max % above 52W Low
       * Market cap filter on/off + value
       * Batch size
       * Start from stock #
       * Delay per stock

9. BUGS FIXED
   - ModuleNotFoundError (SmartApi) — fixed requirements.txt version
   - ValueError on watchlist_data — fixed pandas DataFrame or operator
   - KeyError on r["Sector"] in Signals — fixed with safe .get()

  HOW WATCHLIST FILTER WORKS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Main filter: % Above 52W Low

  Formula: (LTP - 52W Low) / 52W Low x 100

  Example:
    Stock 52W Low = Rs.100
    Today LTP    = Rs.112
    % Above 52W  = 12%    → shows if filter is 10-15%

Status labels:
  🟢 STRONG   = 0% to 12%   → close to yearly low, low risk
  🟡 MODERATE = 12% to 15%  → moderate move, still good
  🟠 WATCH    = above 15%   → already moved up, higher risk

Volume Ratio:
  🔥 Fire     = 1.5x or more above 20-day avg → strong interest, possible breakout
  ⚡ Lightning = 1.2x to 1.5x               → above average interest
  Normal      = below 1.2x                  → nothing unusual

Recommended settings for 1-15% scan:
  Min % = 1
  Max % = 15
  Batch Size = 200
  Start from # = 1 (then 201, 401, 601... for each next batch)

5. Angel One API rate limits
   If too many calls too fast → timeout/skip
   Use delay 0.10-0.12 sec per stock to avoid this
